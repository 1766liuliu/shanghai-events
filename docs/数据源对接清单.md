## 各源对接备注

### 上海本地宝（已实现，`sources/bendibao.py`，两种解析模式）
- 展会(shzhanhui)：`<h3><a>` 文本拼接，正则解析。✅ 可用(25 条),但标题是 SEO 模板、日期/场馆在详情页 → 待抓详情页补全。
- 演出(yanchanghuitime)：表格 `<tr><td>`,已写表格解析器。⚠️ **该页当前是 2024 旧档**,经时效过滤后清零 → 需找到「live 演出排期」正确 URL(或改用文化云)。
- 体育(tiyusaishi)：桌面端 URL 404 → 体育赛事改由久事覆盖,本地宝不再抓体育。

### 久事体育（已实现，`sources/jussevent.py`）
- 结构核实(2026-06)：卡片 `div.flex_index`，标题 `.list_info_title`，日期 `.content_list_bottom_time`，链接 `a[href*=Detail.aspx]`。解析正常(154 条)。
- **关键局限**：首页卡片多为「圆满落幕」往期回顾(日期已过),经时效过滤后基本清零;真正**未来的大型赛事**(F1 2026/大师赛等)在首页「国际赛事」图片栏(链接无文字,难解析)。
- 落地建议:① 解析「国际赛事」图片栏 href → 逐个抓详情页拿名称/日期;或 ② 维护一份「上海年度大型赛事种子表」(F1中国大奖赛/上海劳力士大师赛/斯诺克上海大师赛/环球马术冠军赛/长三角田径钻石赛/上海马拉松),人工确认档期——对家庭提前规划价值最高。

### 重大活动策展源（`sources/curated.py`，核心骨架）
- 人工维护的上海年度重磅活动清单(WAIC/进博会/工博会/上海大师赛/上海马拉松等),带**真实日期**(2026-06 联网核实),`note` 标"待官方确认"者为档期估计。
- 解决了"免费爬虫给不出未来重大活动、且混入过期/不适合内容"的根本问题。每年更新档期即可,直接编辑 `EVENTS`。

### 猫眼演出（已实现，`sources/maoyan.py`）
- 演唱会/剧场/马戏/魔术等商业演出的结构化来源(带真实日期+场馆),Next.js SSR HTML。
- 结构核实(2026-06): 首页热门区 `.hotlist-item` → 名称 `.hotlist-item-name` / 场馆 `.location` / 日期 `.date`。首抓 65 条。
- 局限: ① 卡片走 JS、无 href,详情链接留空;② 城市由猫眼默认控制(当前默认上海),如需强制上海可加 `cityId` cookie;③ 仅"热门"区,非全量,后续可接其分类/分页接口扩量。

### 小红书（暂缓,best-effort 候选）
- UGC 软文、强反爬(签名/滑块/封号)、无结构化字段、需二次内容审核 → 维护成本极高。
- 结论: 演唱会/剧场/马戏/魔术已由猫眼覆盖;小红书仅作"网红遛娃/快闪"发现层,待真有需要再做。

### 内容安全过滤（`pipeline/safety.py`，给孩子用必须有）
- 黑名单关键词(成人展/情趣/性文化…)命中即丢弃。已拦截"上海成人展"等。黑名单按需扩充。

### 时效过滤（`pipeline/freshness.py`）
- 丢弃「有明确日期且早于今天」的活动;无日期但标题年份 < 今年(如"2024演唱会")也丢弃;其余保留。雷达只看未来。

### 文化上海云（优先，`sources/wenhuayun.py` 待抓包）
- 移动优先 SPA，无公开 API。**需在国内网络抓包**：
  1. 打开文化云列表 → 开发者工具 Network/XHR 看翻页请求；
  2. 记录 URL、参数、请求头；
  3. 填进 `wenhuayun.py` 的 `API_URL/PARAMS/HEADERS`，补全 `_parse_item` 字段映射。
- 亲子内容最多、合规高，对接后是核心源。

### 官方权威源（会展平台/体育局/久事）
- 多为服务端渲染 HTML，按 `bendibao.py` 的模式写解析即可，合规风险低，优先于商业源。

### 商业票务（P1）
- 反爬+风控，建议「关注列表 + 低频监控开票状态」，并准备人工兜底；需要时在 `config.py` 配代理。

# 部署与开发指南

## 准备（你本人操作，我无法代办）

1. **注册微信小程序账号**：微信公众平台 → 注册 → 小程序（个人主体免费），拿到 **AppId**。
2. **下载微信开发者工具**（官方）。

## 一、采集层（本机，已可跑）

```bash
cd collector
pip3 install -r requirements.txt
python3 main.py            # 抓取 → 清洗 → 打标 → 写 data/events.json
```

- 验证产物：`data/events.json`。
- 接入云数据库后：`python3 main.py --cloud`（先配 `config.py`）。
- 配置：`cp config.example.py config.py` 后填云开发凭据。

## 二、云后端（微信云开发）

1. 开发者工具开通云开发，记「环境 ID」。
2. 新建集合：`events` `weekly` `favorites` `watch`（权限见 `cloud/README.md`）。
3. `cloud/cloudfunctions/` 三个函数逐个「上传并部署：云端安装依赖」。
4. 给 `weeklyDigest`/`ticketWatch` 配定时触发器。

## 三、小程序前端

1. 开发者工具导入 `miniprogram/`，填 AppId。
2. 环境 ID 填 `app.js` 的 `globalData.env`。
3. 编译预览。

## 四、打通

1. 跑采集层 `--cloud` 把活动写进 `events`；
2. 小程序「发现」即可看到数据；
3. 申请「开票提醒」订阅消息模板，模板 ID 填 `ticketWatch/index.js` 与 `pages/detail/index.js`。

## 五、自动刷新（已配置，每天 8:00 / 20:00）

项目放在 `~/上海家庭活动雷达/`（不放桌面——macOS 禁止后台任务读写桌面/文稿/下载）。

- **触发器**：launchd 任务 `~/Library/LaunchAgents/com.shanghai-radar.refresh.plist`，每天 8:00、20:00 自动跑 `refresh.sh`（重新抓取→过期退出→新增进入→重写 `events.json` + `活动预览.html`）。
- **预览页**：`data/活动预览.html` 已设每 30 分钟自动重载，开着就会更新。
- **手动刷新**：`bash ~/上海家庭活动雷达/refresh.sh`
- **看运行日志**：`data/refresh.log`
- **停用**：`launchctl unload ~/Library/LaunchAgents/com.shanghai-radar.refresh.plist`
- **改时间**：编辑 plist 里的 `StartCalendarInterval` 后 unload 再 load。

> "实时"对活动信息=每天刷新足够;开票/抢票级提醒留待小程序云函数阶段。
